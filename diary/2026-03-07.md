## Laws-only dense retrieval

Tested the first clean hypothesis from the EDA: drop BM25 entirely, use laws corpus only, dense retrieval only.

### What changed from baseline

- **No BM25** — cross-lingual BM25 (English query vs German corpus) was retrieving noise. Removing it entirely improved score.
- **Laws corpus only** — court corpus excluded. Consistent with Talha's finding that adding court dropped his score (laws: 0.01454 → laws+court: 0.01384).
- **Dense (semantic) retrieval only** — bge-m3 embeddings handle the English→German cross-lingual matching.

### Result

| Experiment | LB Score |
|------------|----------|
| Baseline (BM25 + hybrid) | 0.01816 |
| Laws-only dense | 0.02064 |

### What this confirms

- BM25 was a net negative for cross-lingual retrieval. Removing it alone gives a meaningful gain.
- Dense retrieval generalizes across languages well enough to beat the hybrid approach at this stage.
- Laws-only is a sound starting point. Court corpus needs better handling (filtering short rows, chunking) before it adds value.

### What's still broken

- Top-k is still fixed at 10 — val median is 22 citations. Structural recall ceiling remains.
- Query truncation at 2,000 chars still in place.
- No query translation — bge-m3 handles cross-lingual but explicit translation could help the reranker.

## Reranker exploration — cross-lingual mismatch finding

Explored how `bge-reranker-v2-m3` behaves in the actual pipeline. Built `notebooks/reranker_exploration.ipynb` to measure scores across controlled relevance tiers, then added a cross-lingual diagnostic to the EDA notebook.

### Key finding: the cross-encoder reranker is nearly blind to relevant German docs

The pipeline feeds English queries directly into the reranker against German documents. We measured the effect with a controlled experiment — same documents, three query language conditions:

| Pair | Exact-match doc score |
|---|---|
| EN-query / DE-doc (pipeline) | **0.022** |
| DE-query / DE-doc (oracle) | **0.787** |
| EN-query / EN-doc (control) | **0.841** |

The most relevant document scores 0.022 in the actual pipeline — indistinguishable from gibberish (≈0.000016). The language penalty is **+0.764** on the relevant doc and effectively zero on all irrelevant docs.

This is the worst possible failure mode: the model doesn't uniformly depress scores, it specifically suppresses the signal on semantically relevant pairs. The reranker has the knowledge (0.787 with a German query) but cannot bridge the token-level language gap at inference time.

### Why it happens

A cross-encoder sees query + document as raw concatenated tokens. Relevance is scored by attending across query tokens to document tokens. "detention" and "Untersuchungshaft" are completely different token IDs — no surface overlap, no trained cross-lingual attention bridge (fine-tuning was predominantly on English MS-MARCO pairs). The `[CLS]` relevance signal never fires.

Bi-encoders don't have this problem because each text is compressed into a vector independently — language is gone before comparison.

### Fix

Translate queries to German before passing to the reranker. The bi-encoder retrieval step handles English queries fine (cross-lingual embeddings). Only the reranker needs German input. This is based on one controlled toy example — need to validate on real val queries to know the actual recall impact.

### Other reranker observations

- Score distribution across all val queries (Example 3/4 in the notebook): gold citations don't always separate cleanly from hard negatives at a fixed threshold. Score gap between consecutive ranks is more reliable than an absolute cutoff.
- The 0.5 sigmoid threshold is not a reliable separator — score scale shifts per query.

## Query translation experiments

**Notebooks:** [`notebooks/2026-03-07-helsinki.ipynb`](../notebooks/2026-03-07-helsinki.ipynb), [`notebooks/2026-03-07-nllb.ipynb`](../notebooks/2026-03-07-nllb.ipynb)

Tested the translation fix from the reranker exploration: translate English queries to German before passing to the cross-encoder. Bi-encoder retrieval keeps the original English query (bge-m3 handles cross-lingual fine). Two translation models compared.

### Results

| Experiment | LB Score |
|------------|----------|
| Baseline (BM25 + hybrid) | 0.01816 |
| Laws-only dense (no translation) | 0.02064 |
| Laws-only dense + Helsinki `opus-mt-en-de` | **0.03** |
| Laws-only dense + NLLB 600M | 0.018 |

### Key findings

- Helsinki `opus-mt-en-de` (+45% vs laws-only dense, +65% vs baseline) — translation hypothesis confirmed on real LB data.
- NLLB 600M underperforms despite being a larger, higher-quality multilingual model. Possible reasons: NLLB is trained for low-resource languages, EN→DE is a high-resource pair where the specialized Helsinki model has an edge; or translation quality doesn't directly correlate with reranker signal recovery for legal domain text.
- NLLB 1.3B still to be tested.
