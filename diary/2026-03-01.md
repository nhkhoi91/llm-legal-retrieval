## Learnings

**The notebook uses a hybrid retrieval approach:**
- BM25 for exact keyword matching
- `bge-m3` embedding model to filter the top-k candidates, then `bge-reranker-v2-m3` to rerank them
- Reciprocal Rank Fusion (RRF) to combine both results

**What is BM25:**
- A more sophisticated version of TF-IDF, with two key improvements:
  - Diminishing returns on term frequency — a term appearing 1000 times is not 200x more significant than one appearing 5 times
  - Length normalization — a term appearing 5 times in a 2-page document is more relevant than the same term in a 1000-page document, where it could easily be coincidental

## Potential Improvements

- **Text truncation** — the baseline uses `[:text_truncate]`, which cuts strings at a fixed character count. This can split words mid-way and ignores natural boundaries. A better approach would be to split on newlines or by token count instead of raw characters.

- **Court cases embedding** — the baseline skips embedding search for `court_considerations.csv` because the file is too large. We could use FAISS to build a compressed vector index and bring it back into the dense retrieval pipeline.

- **Language mismatch** — the evaluation queries are in English, but the legal documents are likely in German. Translating the queries to German before searching could improve recall significantly.

- **Dynamic top-k** — the baseline always returns a fixed `top_k`. A smarter approach would be to look at the score gap: if the top 3 results score around 0.9 and the 4th drops to 0.3, cut off there instead of returning all k results blindly.
